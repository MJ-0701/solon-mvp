> **Context**: WU-22 §3 β 채택 → 0.3.0-mvp = #1 + #4 + #6 번들. 본 WU 는 #1 의 **detail design only** (구현은 WU-24~26).

## §0. 작업 진행 원칙 (본 WU 한정)

1. **MINIMAL contract** — 각 명령의 "최소 무엇을 하는가" 만 정의. 구체 출력 포맷 / 색상 / 이모지 등 product decision 은 ⚠️ marker 후 사용자 복귀 대기.
2. **3-agent vote** — §1 contract draft 1차안 완성 후 CEO/CTO/CPO 각자 1표. 2/3 PASS → §3 에 결정 기록 + 다음 단계. 2/3 FAIL → 1 retry round. 2nd FAIL → ⚠️ escalate.
3. **사실 작업 vote skip** — 단순 frontmatter 갱신, sha backfill, typo 수정은 sole-author OK (planner.md "When to SKIP 3-agent Vote" 참조).
4. **R-D1 준수** — `~/workspace/solon-mvp/` 는 read-only 로 참조만. 본 WU 의 산출물은 docset 안에만 (구현은 후속 WU 가 templates/scripts/ 에 반영).
5. **§1.5 push 금지** — 21번째 세션은 사용자 부재. local commit 까지만, push 는 사용자 복귀 후 본인 터미널.

## §1. 6개 명령 minimal contract spec (1차 draft, vote 대기)

> 이 §1 의 spec 은 **CEO 1차 draft** 입니다. CTO 가 검토 + 보완 + 자체 vote, CPO 가 5축 평가 + vote, CEO 자체 vote. 3 vote 합산 → 2/3 PASS 면 본 §1 확정 + §3 에 vote_record 기록.

### 1.1 `/sfs status` — 현 상태 1줄 요약 출력

| 항목 | 내용 |
|---|---|
| **Goal** | 현재 sprint / WU / gate 상태를 1줄 요약 (CLI 응답 + 사용자 대시보드 용도) |
| **Input** | (none) — `.sfs-local/` 디렉토리 자동 탐색 |
| **Output (stdout)** | 1 줄 텍스트: `sprint <id> · WU <wu_id> · gate <last_gate>:<verdict> · ahead <N> · last_event <ts>` |
| **Side-effect** | 없음 (read-only) |
| **Error cases** | (a) `.sfs-local/` 부재 → exit 1, stderr "no .sfs-local found, run /sfs start first". (b) `events.jsonl` 손상 → exit 2, stderr "events.jsonl parse error at line <n>". (c) git repo 아님 → exit 3, stderr "not a git repo (sfs requires git for ahead count)". |
| **Exit code** | 0 success / 1 no init / 2 corrupt / 3 no-git / 99 unknown |
| **⚠️ pending product decisions** | 정확한 출력 포맷 (구분자: `·` vs `\|` vs space)? 색상 사용 여부? `last_event` ISO8601 vs 사람 읽기 좋은 포맷? → 사용자 결정 필요 (escalate marker) |

### 1.2 `/sfs start` — 새 sprint 초기화 (= scaffold dir + 4 templates)

| 항목 | 내용 |
|---|---|
| **Goal** | 새 sprint 디렉토리 + 4 파일 (plan/log/review/retro) 자동 scaffold. #6 Sprint cycle CLI helper 와 통합. |
| **Input** | `<sprint-id>` (positional, optional). 미지정 시 자동 생성: `<YYYY-W>-sprint-<N>` (W=ISO week, N=주 안의 sprint 번호). |
| **Output (stdout)** | 다중 줄: `created: sprints/<sprint-id>/` + 생성 파일 4개 list. |
| **Side-effect** | (a) `sprints/<sprint-id>/` 디렉토리 생성. (b) `plan.md`/`log.md`/`review.md`/`retro.md` 4 파일 templates 복사. (c) `.sfs-local/events.jsonl` 에 entry append (`{"ts":..., "type":"sprint_start", "sprint_id":..., "by":"sfs-start"}`). |
| **Error cases** | (a) 동명 sprint 존재 → exit 1, stderr "sprint <id> already exists, use --force or pick another id". (b) templates 부재 (`templates/sprint-templates/` not found) → exit 4, stderr. (c) 권한 부족 → exit 5. |
| **Exit code** | 0 success / 1 conflict / 4 templates-missing / 5 permission / 99 unknown |
| **⚠️ pending product decisions** | sprint-id auto naming: ISO week (`2026-W17-sprint-1`) vs date (`2026-04-25-sprint-1`)? 사용자 docset 의 기존 sprint 명명 규칙과 정합 필요. |

### 1.3 `/sfs plan` — 현 sprint 의 plan.md 진입 (open / scaffold / append)

| 항목 | 내용 |
|---|---|
| **Goal** | 현 sprint 의 `plan.md` 를 사용자 편집 진입 상태로 만든다. 미존재 시 template 으로 신설. 존재 시 frontmatter `phase: plan` 갱신. |
| **Input** | (none — 현 sprint 컨텍스트는 `.sfs-local/current-sprint` 에서 추출) |
| **Output (stdout)** | `plan.md ready: <path>` 1 줄. (실제 편집은 사용자 에디터 책임 — `/sfs plan` 은 파일 준비만). |
| **Side-effect** | (a) `plan.md` 미존재면 신설 (template 적용). (b) frontmatter `phase: plan` + `last_touched_at` 갱신. (c) `events.jsonl` append (`{"type":"plan_open", ...}`). |
| **Error cases** | (a) 활성 sprint 없음 → exit 1, stderr "no active sprint, run /sfs start first". (b) `current-sprint` 파일 손상 → exit 2. |
| **Exit code** | 0 / 1 no-sprint / 2 corrupt / 99 |
| **⚠️ pending product decisions** | 명령이 에디터 자동 launch 해야 하는가 (`$EDITOR plan.md`) vs 단순 파일 준비만? 사용자 워크플로 (vim / VS Code / Cursor) 에 따라 다름. |

### 1.4 `/sfs review` — 현 sprint review (gate verdict 기록 진입)

| 항목 | 내용 |
|---|---|
| **Goal** | 현 sprint 의 `review.md` 작성 진입 + gate verdict 기록 가능 상태로 만든다. |
| **Input** | (optional) `--gate <gate-id>` (예: G2 / G4) — verdict 기록 대상 gate. 미지정 시 마지막 active gate. |
| **Output (stdout)** | `review.md ready: <path> | gate <gate-id> awaiting verdict`. |
| **Side-effect** | (a) `review.md` frontmatter `phase: review` + `gate_id` 갱신. (b) `events.jsonl` append (`{"type":"review_open", "gate_id":..., ...}`). |
| **Error cases** | (a) 활성 sprint 없음 → exit 1. (b) 알 수 없는 gate-id → exit 6, stderr "unknown gate <id>, valid: G0/G1/G2/G4". |
| **Exit code** | 0 / 1 no-sprint / 6 unknown-gate / 99 |
| **⚠️ pending product decisions** | gate id schema 가 docset 본문 (G0/G1/G2/G4 — `00-intro.md` 또는 본문 어딘가) 에 정의돼 있는가? 검증 필요. spec 안 되어 있으면 본 명령이 정의해야 함 (별도 의미 결정). |

### 1.5 `/sfs decision` — mini-ADR 신설

| 항목 | 내용 |
|---|---|
| **Goal** | `decisions/<id>-<kebab>.md` mini-ADR 자동 생성 (#5 ADR helper 와 통합 — 0.3.x patch). |
| **Input** | `<title>` (positional, required) — 결정 1줄 제목. (optional) `--id <override-id>` — id 수동 지정. |
| **Output (stdout)** | `decision created: decisions/<id>-<kebab>.md` 1 줄. |
| **Side-effect** | (a) `decisions/` 디렉토리 (없으면 생성) + 새 ADR 파일 (frontmatter + 5 섹션 template). id 자동 부여 (가장 큰 기존 id + 1, 시작은 `0001`). (b) `events.jsonl` append (`{"type":"decision_created", "decision_id":..., ...}`). |
| **Error cases** | (a) `<title>` 미지정 → exit 7, stderr usage. (b) `--id` 충돌 → exit 1, stderr "decision id <id> already exists". (c) template 부재 → exit 4. |
| **Exit code** | 0 / 1 conflict / 4 template-missing / 7 missing-arg / 99 |
| **⚠️ pending product decisions** | mini-ADR 템플릿 5 섹션 = (Context / Decision / Alternatives / Consequences / References) — 사용자 docset 의 cross-ref-audit §4 W10 TODO 형식과 정합 여부 확인 필요. |

### 1.6 `/sfs retro` — 현 sprint retrospective 진입

| 항목 | 내용 |
|---|---|
| **Goal** | 현 sprint 의 `retro.md` 작성 진입 + sprint close 준비. |
| **Input** | (optional) `--close` — retro 작성 후 sprint 을 `closed` 상태로 마킹. |
| **Output (stdout)** | `retro.md ready: <path>` (+ `sprint closed: <id>` if `--close`). |
| **Side-effect** | (a) `retro.md` frontmatter `phase: retro` 갱신. (b) `--close` 시 `current-sprint` 클리어 + sprint frontmatter `status: closed` + `closed_at`. (c) `events.jsonl` append. |
| **Error cases** | (a) 활성 sprint 없음 → exit 1. (b) `--close` 인데 review.md 미작성 → exit 8, stderr "review.md required before close (run /sfs review first)". |
| **Exit code** | 0 / 1 no-sprint / 8 review-missing / 99 |
| **⚠️ pending product decisions** | `--close` 가 자동 git commit 도 해야 하는가? §1.5 (no auto push) 와 별개로 commit 자동화는 사용자 워크플로 결정 필요. |

## §2. 공통 모듈 / 의존성

- **`.sfs-local/current-sprint`** — 현 활성 sprint id 1줄 텍스트. 모든 명령이 이걸로 컨텍스트 파악.
- **`.sfs-local/events.jsonl`** — append-only event log. 모든 명령이 entry 1개 추가 (#4 schema 표준화 WU-27 에서 schema 확정).
- **공통 helper** — `sfs-common.sh`: (a) `read_current_sprint()`, (b) `append_event()`, (c) `validate_sfs_local()`, (d) error exit helper.
- **외부 의존성** — git (status/log/rev-list), jq (events.jsonl 읽기/쓰기 — 권장 but optional). bash 4+.
