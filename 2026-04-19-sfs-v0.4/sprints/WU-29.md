## §2. Finding #2 — macOS tac fallback

**Before**: `sfs-common.sh` 4 곳 (line 88 / 97 / 105 / 289) 이 `tac` 직접 호출. macOS 기본 환경 = tac 없음 → silent fail (events.jsonl scan 결과 빈 stdout, 모든 read_last_* helper 가 무력).

**After**: helper `reverse_lines()` 신설 + 4 곳 교체.

```bash
# Helper signature (sfs-common.sh, CONSTANTS 블록 직후 추가)
# reverse_lines — 파일 내용을 역순 출력 (행 단위)
#   $1 = 파일 경로
# Returns: 0 (파일 없으면 빈 출력 + rc=0, callers 가 grep -m1 ... || true 패턴이라 OK)
# 우선순위: tac (gnu coreutils) > gtac (homebrew gnu) > tail -r (BSD/macOS) > awk fallback
reverse_lines() {
  local f="${1:-}"
  [[ -f "${f}" ]] || return 0
  if command -v tac >/dev/null 2>&1; then
    tac "${f}"
  elif command -v gtac >/dev/null 2>&1; then
    gtac "${f}"
  elif tail -r </dev/null >/dev/null 2>&1; then
    # BSD tail (macOS) — `-r` reverse mode
    tail -r "${f}"
  else
    awk '{a[NR]=$0} END {for (i=NR;i>0;i--) print a[i]}' "${f}"
  fi
}
```

**4 호출자 변경** (모두 동일 패턴):
- `read_current_wu` (line 88): `tac "${SFS_EVENTS_FILE}" 2>/dev/null` → `reverse_lines "${SFS_EVENTS_FILE}"`
- `read_last_gate` (line 97): 동상
- `read_last_gate_verdict` (line 105): 동상
- `infer_last_gate_id` (line 289): 동상 (단 trailing `|| true` + `return 0` 보존, 24th 49번째 amazing-determined-gates 의 pipefail bug fix 정합)

**검증**: sandbox `/tmp/wu29-smoke-tac/` 안에서 `PATH=` (tac 비활성) + `command -v tail` (BSD/macOS) 시뮬 → events.jsonl 4-event mock 로 4 helper 모두 정상 reverse stdout 확인. `2>/dev/null` redirect 는 reverse_lines 안에서 처리.

**근거**: codex review finding #2 "macOS tac fallback" verbatim. `tail -r` 는 macOS BSD tail 의 표준 reverse flag (POSIX 외 확장이지만 macOS 기본 동작 OK). awk 폴백은 마지막 안전망 — 매우 큰 events.jsonl 일 경우에도 메모리 OK (현 시점 events.jsonl 평균 < 1MB).
