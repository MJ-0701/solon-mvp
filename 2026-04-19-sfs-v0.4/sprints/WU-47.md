# WU-47 — Context Router Same-Version Repair Audit

## §0. Findings

- Fresh `sfs init --layout thin --yes` installs `.sfs-local/context/` correctly:
  `_INDEX.md`, `kernel.md`, command modules, and `policies/mutex.md`.
- Packaged user entry docs are small routers: Codex Skill is 36 lines and
  directs agents to read `kernel.md`, `_INDEX.md`, then the routed module.
- Existing dev repo self-dogfood lagged behind: root `.agents/skills/sfs/SKILL.md`
  was still a 575-line monolith and `.sfs-local/context/` was absent.
- `sfs upgrade` same-version fast path repaired routed command/policy modules
  but missed the core `_INDEX.md` and `kernel.md` files when context was absent.

## §1. Changes

- `upgrade.sh` now repairs `_INDEX.md` and `kernel.md` alongside routed
  modules when a project already reports the latest version.
- `verify_context_router_targets` fails closed if `_INDEX.md` or `kernel.md`
  remains missing.
- Product release verification now checks packaged `_INDEX.md` and `kernel.md`
  before validating routed modules.
- This dev repo now uses the same 36-line Codex SFS router as product projects,
  with `.sfs-local/context/` populated from the distribution template.

## §2. Verification

- Fresh thin init context list matched the distribution template.
- Same-version upgrade smoke with missing context restored every router file,
  including `_INDEX.md` and `kernel.md`.
- `diff -qr` passed for both temp restored context and dev repo context against
  `solon-mvp-dist/templates/.sfs-local-template/context`.
- `bash -n solon-mvp-dist/upgrade.sh scripts/verify-product-release.sh`.
