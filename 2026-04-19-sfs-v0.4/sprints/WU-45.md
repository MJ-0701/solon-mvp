# WU-45 — Dev HQ Backend Architecture Evolution Ladder

## §0. Intent

Make the Dev division's backend architecture behavior explicit in `/sfs
implement` so small projects stay simple while growing projects get timely
architecture transition guidance.

## §1. Contract Added

- MVP and small projects default to clean layered monolith.
- Backend work beyond the initial MVP applies CQRS at the application boundary,
  even when the system still has one database.
- Hexagonal Architecture is introduced as a transition candidate when domain
  seams, integrations, release cadence, or test seams become visibly
  independent. Refactor only after user acceptance.
- MSA is introduced only when independent deployment, scaling, ownership,
  resilience, or blast-radius isolation requires service boundaries. Refactor
  only after explicit approval.

## §2. Implementation Summary

- Added the ladder to the active Codex SFS Skill and packaged Codex Skill
  template.
- Added routed implement context and implementation worker persona rules.
- Expanded `implement.md` and `log.md` templates with architecture mode,
  approval, trigger evidence, and transition evidence fields.
- Updated Dev division SSoT, README, GUIDE, glossary, and release notes.
