## §3.1 Solon-wide executor convention (신규 도입, 본 sub-task 1 핵심)

**Scope**: 본 convention 은 `/sfs loop` 단독 적용이 아니라 **Solon 시스템 전체** (sfs CLI 통합 레이어). 현재 LLM 호출하는 sub-cmd = `/sfs loop` 단독 (본 WU). 미래 잠재 consumer = `/sfs review --auto-judge` / `/sfs plan --suggest` / `/sfs retro --summarize` 등 (0.5.0-mvp+ 후속, 본 WU 범위 밖).

### Convention spec

- **Global flag**: `--executor <profile|cmd>` — 모든 sfs sub-cmd 인식.
- **Env var**: `SFS_EXECUTOR` — system default. CLI flag > env > hardcoded fallback.
- **Hardcoded fallback**: `claude` (= `claude -p --dangerously-skip-permissions`).
- **Shared helper**: `sfs-common.sh::resolve_executor()` — named profile dispatch + custom string passthrough.

### Named profile registry (sub-task 6 구현 대상)

| profile | resolved cmd | non-interactive flag | auto-approve flag | API key env |
|---|---|---|---|---|
| `claude` (default) | `claude -p --dangerously-skip-permissions` | `-p` | `--dangerously-skip-permissions` | `ANTHROPIC_API_KEY` |
| `gemini` | `gemini -p --yolo` | `-p` | `--yolo` | `GEMINI_API_KEY` |
| `codex` | `codex exec --full-auto` | `exec` | `--full-auto` | `OPENAI_API_KEY` |
| `<custom string>` | passthrough as-is | — | — | (사용자 환경 책임) |

### Helper signature (구현 = sub-task 6, 본 sub-task 1 = spec 명세만)

```sh
# sfs-common.sh (sub-task 6 추가 대상)
resolve_executor() {
  local executor="${1:-${SFS_EXECUTOR:-claude}}"
  case "$executor" in
    claude) echo "claude -p --dangerously-skip-permissions" ;;
    gemini) echo "gemini -p --yolo" ;;
    codex)  echo "codex exec --full-auto" ;;
    *)      echo "$executor" ;;   # custom string passthrough
  esac
}
```
