# WU-38 — Release Channel Verification Hotfix

## §0. Problem

`0.5.60-product` was pushed to stable, Homebrew tap, and Scoop bucket, but the
actual Homebrew tap clone under `/opt/homebrew/Library/Taps/...` was still at
`0.5.57-product`. The release was real remotely, but the local user upgrade
path could still stop at an older formula.

## §1. Fix

- `sfs upgrade` now explicitly fast-forwards the Solon Homebrew tap clone before
  running package upgrade.
- Homebrew self-upgrade now uses the fully qualified formula
  `MJ-0701/solon-product/sfs` instead of ambiguous `sfs`.
- Added `scripts/verify-product-release.sh` to verify product tag, Homebrew
  remote formula, local Homebrew tap freshness, Scoop remote manifest, archive
  hashes, and installed `sfs version --check`.
- Release docs now require the verifier before reporting product deploy
  complete.

## §2. Verification

- Actual local Homebrew tap clone was fast-forwarded to `97ceb6c`.
- Actual global Homebrew `sfs` upgraded to `0.5.60-product`.
- Scoop remote raw manifest checked at `0.5.60-product`; local mac has no
  Scoop CLI, so Windows installed-app smoke is not available in this session.
- Pending after commit/release: run verifier for `0.5.61-product`.
