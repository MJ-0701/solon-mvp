# WU-43 — sfs start Next-Action Guidance

## §0. Goal

Make `sfs start` behave like a useful first step:

- preserve adapter stdout as SSoT.
- do not create or imply `report.md` at start time.
- still show the user the next Solon action, normally `sfs brainstorm`.

## §1. Acceptance Criteria

1. `sfs start "<goal>"` prints scaffolded files plus one readable `next: sfs brainstorm "<goal>"` line.
2. `sfs start` without a goal prints a placeholder brainstorm next action.
3. Routed agent context explicitly says bash-first means no artifact refinement, not no Next.
4. Existing `sfs-profile.sh` orphan file is inspected but excluded from this WU because it is an unwired new public command.
