## §2. 의존성 그래프

```
#1 (sfs slash, dispatcher)  ←  거의 모든 후보의 enabler
  ├─→ #4 (events.jsonl schema)  ─→  #8 (doctor 입력)
  ├─→ #6 (sprint helper)
  ├─→ #5 (ADR helper)
  └─→ #7 (신규 어댑터, slash 안정 후)

#2 (upgrade.sh)  ─┬─  공통 dry-run 패턴
                  └→  #3 (uninstall.sh)
                       └→  #8 (doctor 의 VERSION drift 검사)
```

핵심 관찰:
- **#1 이 모든 후보의 백본**. #1 의 minimal contract 가 안정되면 나머지가 가벼워짐.
- **#4 + #6 + #5 는 #1 의 부속**. #1 와 같은 release 묶음 권장.
- **#2 + #3 은 install/upgrade/uninstall 트리오 완성**. 같이 묶으면 효율.
- **#7 은 #1 안정 이후 별도 release** (OSS 도달 확장 마케팅 타이밍과 분리 가능).
- **#8 은 마지막**. 앞 후보들이 만든 산출물을 진단하는 게 본질.

### 옵션 α — Sequential (사용자 우선순위 그대로 1개씩)

```
0.3.0-mvp = #1
0.3.1-mvp = #2
0.3.2-mvp = #4
0.3.3-mvp = #6
0.3.4-mvp = #5
0.3.5-mvp = #3
0.3.6-mvp = #7
0.4.0-mvp = #8 + retrospective consolidation
```

- **Pros**: release 1건당 risk 작음, 사용자가 매번 검수 가능.
- **Cons**: 8 release = upgrade 빈도 부담. patch release 안에 같이 가도 될 작은 항목까지 분리.

### 옵션 β — Themed bundles ✅ **CHOSEN**

```
0.3.0-mvp = #1 + #4 + #6              (slash command core + 부속)         ← 다음 release
0.3.x patch = #5                       (decision helper, ADR)
0.4.0-mvp = #2 + #3                   (install lifecycle 완성: upgrade + uninstall)
0.5.0-mvp = #7                         (어댑터 확장, OSS 트랙)
0.6.0-mvp = #8                         (doctor, 종합 진단)
```

- **Pros**: 의존성 그래프 따라 자연스러움. release 4-5건 적당. 각 release 가 사용자 가치 명확.
- **Cons**: 0.3.0-mvp 가 좀 큼 (#1 의 L 사이즈). 더 잘게 자르고 싶으면 α.

**β 에 따른 sprint roadmap (신규)**:
- WU-23 = #1 sfs slash command **detail design** (6 명령 minimal contract spec). WU22-D2 결정 선행.
- WU-24~26 = #1 구현 (명령군 3 분할 예상: status/start · plan/review · decision/retro)
- WU-27 = #4 events.jsonl schema 표준화 (0.3.0-mvp 번들)
- WU-28 = #6 Sprint cycle CLI helper (0.3.0-mvp 번들)
- WU-29 = #5 ADR helper (0.3.x patch)
- 0.3.0-mvp cut: WU-23~28 묶음 완료 시점
- 0.4.0-mvp ~ 0.6.0-mvp: 별도 WU 시리즈로 분기 (roadmap 우측)

### 옵션 γ — Big bang (8개 모두 0.4.0-mvp 한 방)

- **Pros**: release 1건. consumer upgrade 1번.
- **Cons**: scope 너무 큼 (총 effort 6-8주). 중간에 사용자 결정 사안 누적되면 막힘. **권장하지 않음**.
