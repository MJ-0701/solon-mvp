## §1. `/sfs status` 구현 spec (WU22-D4 적용)

### 1.1 Output 형식 확정

```
sprint <id> · WU <wu_id> · gate <last_gate>:<verdict> · ahead <N> · last_event <ISO8601_ts>
```

- **구분자**: `·` (U+00B7 middle dot, sed/awk 호환).
- **last_event**: ISO8601 with TZ (`2026-04-25T22:59:00+09:00`).
- **색상**: `--color=auto/always/never` CLI flag.
  - `auto` (default): stdout 이 tty 면 색상 on, redirect 시 off.
  - `always`: 강제 on.
  - `never`: 강제 off.
- **색상 대상** (auto/always 시):
  - `<id>`, `<wu_id>`: 기본 (no color).
  - `<verdict>`: pass=green / partial=yellow / fail=red (gates.md §2 정합).
  - `<N>` (ahead): 0=기본 / >0=cyan.

### 1.2 Bash 골격 (sfs-status.sh draft)

```sh
#!/usr/bin/env bash
# .sfs-local/scripts/sfs-status.sh
set -euo pipefail
source "$(dirname "${BASH_SOURCE[0]}")/sfs-common.sh"

# CLI parse
COLOR_MODE="auto"
while [[ $# -gt 0 ]]; do
  case "$1" in
    --color=*) COLOR_MODE="${1#*=}"; shift ;;
    --color) COLOR_MODE="$2"; shift 2 ;;
    -h|--help) usage_status; exit 0 ;;
    *) echo "unknown arg: $1" >&2; exit 99 ;;
  esac
done

# Validate .sfs-local
validate_sfs_local || exit $?   # 1=no init, 2=corrupt

# Gather state
SPRINT_ID="$(read_current_sprint)"   # may be empty (no active sprint)
WU_ID="$(read_current_wu)"           # may be empty
LAST_GATE="$(read_last_gate)"        # may be empty
VERDICT="$(read_last_gate_verdict)"
AHEAD="$(git_ahead_count)"           # 3=no-git
LAST_EVENT_TS="$(read_last_event_ts)"

# Render with color
render_status_line "${COLOR_MODE}" "${SPRINT_ID}" "${WU_ID}" "${LAST_GATE}" "${VERDICT}" "${AHEAD}" "${LAST_EVENT_TS}"
```

### 1.3 Error / exit codes (WU-23 §1.1 그대로)

| Exit | 의미 | stderr |
|:-:|---|---|
| 0 | success | (none) |
| 1 | `.sfs-local/` 부재 | `no .sfs-local found, run /sfs start first` |
| 2 | events.jsonl 손상 | `events.jsonl parse error at line <n>` |
| 3 | git repo 아님 | `not a git repo (sfs requires git for ahead count)` |
| 99 | unknown | `<original error>` |

## §2. `/sfs start` 구현 spec (WU22-D5 적용)

### 2.1 Sprint-id auto naming 확정

```
default pattern: <YYYY-Wxx>-sprint-<N>
```

- `<YYYY-Wxx>` = ISO 8601 week (`date +%G-W%V`).
- `<N>` = 해당 ISO week 안의 sprint 번호 (1부터, 충돌 시 +1).
- 사용자 명시 인자: `/sfs start [<sprint-id>]` — 미지정 시 auto.

**예시**: 2026-04-25 (토, ISO week 17) → `2026-W17-sprint-1`. 같은 주 두 번째 = `2026-W17-sprint-2`.

### 2.2 Bash 골격 (sfs-start.sh draft)

```sh
#!/usr/bin/env bash
# .sfs-local/scripts/sfs-start.sh
set -euo pipefail
source "$(dirname "${BASH_SOURCE[0]}")/sfs-common.sh"

# CLI parse
SPRINT_ID=""
FORCE=false
while [[ $# -gt 0 ]]; do
  case "$1" in
    --force) FORCE=true; shift ;;
    -h|--help) usage_start; exit 0 ;;
    -*) echo "unknown flag: $1" >&2; exit 99 ;;
    *) SPRINT_ID="$1"; shift ;;
  esac
done

# Auto-generate if missing
if [[ -z "${SPRINT_ID}" ]]; then
  SPRINT_ID="$(generate_sprint_id_iso_week)"
fi

# Conflict check
SPRINT_DIR=".sfs-local/sprints/${SPRINT_ID}"
if [[ -d "${SPRINT_DIR}" ]] && [[ "${FORCE}" != "true" ]]; then
  echo "sprint ${SPRINT_ID} already exists, use --force or pick another id" >&2
  exit 1
fi

# Templates check
TEMPLATES_DIR=".sfs-local/sprint-templates"
[[ -d "${TEMPLATES_DIR}" ]] || { echo "templates not found: ${TEMPLATES_DIR}" >&2; exit 4; }

# Scaffold
mkdir -p "${SPRINT_DIR}"
for f in plan log review retro; do
  cp "${TEMPLATES_DIR}/${f}.md" "${SPRINT_DIR}/${f}.md"
done

# events.jsonl + current-sprint
append_event "sprint_start" "{\"sprint_id\":\"${SPRINT_ID}\",\"by\":\"sfs-start\"}"
echo "${SPRINT_ID}" > .sfs-local/current-sprint

echo "created: ${SPRINT_DIR}/"
echo "  - plan.md"
echo "  - log.md"
echo "  - review.md"
echo "  - retro.md"
```

### 2.3 Sprint templates 4 (신설)

`.sfs-local/sprint-templates/` 하위:
- **`plan.md`** — frontmatter (`phase: plan`, `gate_id: G1`) + sections (요구사항·AC·범위)
- **`log.md`** — Do 단계 작업 로그 (frontmatter `phase: do`)
- **`review.md`** — frontmatter (`phase: review`, `gate_id: G2|G3|G4`) — `/sfs review --gate <id>` 진입 시 갱신
- **`retro.md`** — frontmatter (`phase: retro`, `gate_id: G5`) — `/sfs retro --close` 진입 시 마킹

### 2.4 Error / exit codes (WU-23 §1.2 그대로)

| Exit | 의미 | stderr |
|:-:|---|---|
| 0 | success | (none) |
| 1 | sprint id 충돌 | `sprint <id> already exists, use --force or pick another id` |
| 4 | templates 부재 | `templates not found: .sfs-local/sprint-templates` |
| 5 | 권한 부족 | (system error) |
| 99 | unknown | (original error) |
