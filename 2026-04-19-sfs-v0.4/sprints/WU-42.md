# WU-42 — Windows Scoop Command Docs Alignment

## §0. Goal

Make Windows onboarding consistent across all public docs:

- PowerShell/cmd examples use `sfs.cmd ...`.
- Git Bash/WSL/Mac examples use `sfs ...`.
- Source installer examples are clearly marked as fallback, not the recommended
  Windows path.

## §1. Acceptance Criteria

1. Beginner guide Windows install/check/init/upgrade/help snippets use `sfs.cmd`.
2. README Quickstart, Upgrade, and Version Check sections split Windows and Mac/Git Bash commands.
3. GUIDE troubleshooting mirrors the same command split.
4. Scoop packaging README and manifest notes use `sfs.cmd` in Windows examples.

## §2. Verification

- `git diff --check`
- `python3 -m json.tool solon-mvp-dist/packaging/scoop/sfs.json.template`
- Audited PowerShell fenced blocks in README, BEGINNER-GUIDE, GUIDE, and packaging/scoop README:
  Windows direct shell examples now use `sfs.cmd`; source `install.ps1`/`upgrade.ps1`
  examples are explicitly fallback paths.
