# WU-19 — Phase 1 MVP W0 Executable Scripts

## 배경

WU-18 의 `PHASE1-MVP-QUICK-START.md §2` 는 100 줄 bash 블록을 사용자가 터미널에
복붙하는 방식. "MVP 빠르게" 목적에 부합하지만 복붙 중 실수 / macOS vs Linux `sed`
차이 / 환경변수 누락 등 실수 가능. 이것을 **executable .sh 파일 2 개**로 분리하면:

- **setup-w0.sh**: 환경변수 (또는 인자) 3 개 입력 → repo 생성 + templates cp + placeholder 치환 + 3 commit + push 자동
- **verify-w0.sh**: W0 exit 체크리스트 자동 검증 (IP 경계 git ls-files grep, commit 수, .sfs-local 구조, placeholder 잔여)

## 스코프 (명시적 제한)

### 포함

1. `setup-w0.sh` — idempotent, macOS/Linux 호환, set -euo pipefail, 3 환경변수 입력 (`PROJECT_NAME`, `SOLON_DOCSET`, `WORKSPACE`).
2. `verify-w0.sh` — readonly, W0 exit 5 체크 자동화 + 잔여 placeholder 리포트.
3. `PHASE1-MVP-QUICK-START.md` §2 간소화 (스크립트 호출 1 줄) + §6 간소화 (`verify-w0.sh` 호출 1 줄).
4. `phase1-mvp-templates/README.md` 에 스크립트 2개 설명 추가.

### 제외

- Stack preset (Next.js 15 skeleton 등) — 사용자 선호 분기 (→ 원칙 2 회색 영역) 회피. 사용자가 `npm create next-app` 등 선호 도구로 직접 실행.
- 권한 모델 / DB schema / 현금영수증 API 어댑터 skeleton — G0 브레인스토밍에서 결정해야 할 영역 (원칙 2 엄격 적용).
- Claude Code 플러그인 자동 설치 — `plugin-wip-skeleton/INSTALL-GUIDE.md` 이 이미 수동 1-liner 제공.

## 제약 / 규율

- **원칙 2**: 스크립트 로직 = QUICK-START §2 bash 블록 그대로 이식. 해석 변경 없음. placeholder 치환도 `<PROJECT-NAME>` / `<DATE>` 만 (WU-18 에서 이미 결정된 범위).
- **IP 경계**: 스크립트 내부에 Solon 경로 하드코딩 없음. `SOLON_DOCSET` 환경변수로 입력받음.
- **FUSE bypass 호환**: 스크립트는 사용자 Mac 에서 실행되므로 FUSE 제약 없음. 사용자 터미널 = 정상 git 자격.
- **push 포함**: 사용자 지시 "git 자동화 당분간 ON" 에 따라 setup-w0.sh 가 `git push -u origin main` 포함. 단 실패 시 (자격 없음 등) 사용자가 수동 재시도 가능하게 exit code 0 유지.
